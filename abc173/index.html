C - H and V

最近のCは難しいな、と思いつつも書いてあることをやるだけ。

2の累乗の全探索はbit演算を用いる方法もありますが、自分はpythonなのでitertoolsを使う方法が好きです。

格子の入力が文字なのでmap(int,input().split())をしないように注意。

全探索の範囲を2**hではなく、hにしてしまったため微ロス。

コード
from itertools import product

h, w, k = map(int, input().split())

h_kumiawase = list(product([1, 0], repeat=h))
w_kumiawase = list(product([1, 0], repeat=w))

C = [list(input()) for _ in range(h)]
# C[h][w]


def calc(h_kumi, w_kumi):
    cnt = 0
    for i in range(h):
        if h_kumi[i] == 1:
            continue
        for j in range(w):
            if w_kumi[j] == 1:
                continue
            if C[i][j] == ".":
                continue
            cnt += 1
    return cnt


ans = 0
for i in range(1 << h):
    h_kumi = h_kumiawase[i]
    for j in range(1 << w):
        w_kumi = w_kumiawase[j]

        if calc(h_kumi, w_kumi) == k:
            ans += 1

print(ans)

A.sort(reverse=True)
ans = 0

解説AC。 久しぶりのグラフ問題なので可能ならば解きたい、と思う。 木であることを確認。(辺の数はN-1、閉路なし) 区間に対して計算を行うタイプの問題。(順番を変えればうまくできるかな？) 連結成分の数を答えるらしい…です。( ﾟдﾟ)

ここで思考停止、連結成分をうまく扱う構造を自分はあまり知らないです。 連結と言われるとUnion-Findが浮かぶが、今回の問題ではN=2*(10**5)なので厳しそう。

“Fだし、諦めよう"となりました。

コンテスト後にTwitterを見ていると、木構造だと連結成分の数がかんたんに求まるようでした(閉路がなければ簡単に求められる)。 この方法は見たことがあったので、覚えていなかった自分が悪いです。悲しいです。 Fの解説についてはmaspyさんのHPが素晴らしいのでそちらをご参照ください。

コード

解説AC

n = int(input())
graph = [[] for _ in range(n)]

edge_sum = 0
for _ in range(n - 1):
    u, v = map(int, input().split())
    if u > v:
        u, v = v, u
    edge_sum += u * (n - v + 1)

dot_sum = n * (n + 1) * (n + 2) // 6
print(dot_sum - edge_sum)
